Helm Chart

Helm Chart: Upgrade.

Upgrade safely with pre-checks, risk controls, rollout verification, and rollback awareness.

Upgrade

Helm upgrades can replace runtime pod templates, change PVC requests, change ServiceMonitor labels, or update image tags. Always diff and dry-run before applying.

RiskWhat Can HappenMitigation
Values driftChanging values accidentally disables route, monitoring, or storage topology.Keep values in Git and compare with helm get values.
Pod restartStateful workloads restart and capacity is reduced during rollout.Upgrade only when all nodes are healthy and no repair is running.
PVC resizePVC request changes may require pod restart or storage-class expansion support.Check allowVolumeExpansion and verify PVC capacity after rollout.
Rollback mismatchRolling back chart templates may not roll back data format changes.Confirm runtime release notes before rollback.
 
export LI9_NS=li9-s3-helm
helm repo update
helm get values li9-s3 -n "${LI9_NS}" > current-values.yaml
helm template li9-s3 li9-s3/li9-s3-storage -n "${LI9_NS}" -f values-li9-s3.yaml > next-rendered.yaml
kubectl apply --dry-run=server -f next-rendered.yaml
helm upgrade li9-s3 li9-s3/li9-s3-storage \
  --namespace "${LI9_NS}" \
  --values values-li9-s3.yaml \
  --wait \
  --timeout 20m
helm history li9-s3 -n "${LI9_NS}"
oc get pods,pvc -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3

Post-Upgrade Verification

Confirm that the rendered chart still matches the intended control plane and that no operator-managed installation is present in the same namespace. Helm installs must be independent from OLM installs to avoid two controllers managing the same runtime resources.

helm status li9-s3 -n "${LI9_NS}"
helm get values li9-s3 -n "${LI9_NS}" --all
oc get pods,statefulset,deploy,pvc,svc,route -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3
oc get subscription,csv -A | grep li9-s3 || true
oc get events -n "${LI9_NS}" --field-selector type=Warning | grep li9-s3 || true

Rollback Guidance

helm rollback can restore templates and values, but it cannot reverse data-format migrations. Use rollback only when the target runtime version is marked downgrade-compatible. If the issue is a bad value, prefer a forward helm upgrade that corrects the value.
