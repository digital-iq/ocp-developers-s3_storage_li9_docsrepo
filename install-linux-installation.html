Linux Packages

Linux Packages: Installation.

Run the production installation procedure with copy-paste commands and field references.

Installation

1. Prepare OS resources

Create users, directories, permissions, firewall rules, and TLS directories on every node.

2. Install packages

Install RPM or DEB packages from online or internal repositories.

3. Write service configuration

Use one cluster config with node-specific IDs.

4. Bootstrap, join, and start

Initialize the first node, join the remaining nodes, and enable systemd units on all nodes.

5. Verify S3 and failure behavior

Write from an external client while one node is stopped.

1. Prepare Each Host

sudo useradd --system --home-dir /var/lib/li9/s3-storage --shell /usr/sbin/nologin li9-s3 || true
sudo install -d -o li9-s3 -g li9-s3 -m 0750 /var/lib/li9/s3-storage /var/log/li9/s3-storage
sudo install -d -o root -g li9-s3 -m 0750 /etc/li9/s3-storage /etc/li9/s3-storage/tls
sudo firewall-cmd --add-port=9000/tcp --permanent || true
sudo firewall-cmd --add-port=9001-9010/tcp --permanent || true
sudo firewall-cmd --reload || true

2. Install Packages

Use the Linux package release as the connected source, or replace LI9_LINUX_RELEASE_BASE with your internal mirror URL in disconnected environments.

export LI9_NATIVE_VERSION=1.1.0-alpha.14
: "${LI9_LINUX_RELEASE_BASE:?set to the Linux package release or internal mirror URL}"

curl -fL "${LI9_LINUX_RELEASE_BASE}/li9-s3-storage-1.1.0.alpha.14-1.x86_64.rpm" -o li9-s3-storage-1.1.0.alpha.14-1.x86_64.rpm
curl -fL "${LI9_LINUX_RELEASE_BASE}/li9-s3-storage_1.1.0.alpha.14-1_amd64.deb" -o li9-s3-storage_1.1.0.alpha.14-1_amd64.deb
curl -fL "${LI9_LINUX_RELEASE_BASE}/checksums.txt" -o checksums.txt
sha256sum -c checksums.txt

if command -v dnf >/dev/null 2>&1; then
  sudo dnf install -y ./li9-s3-storage-1.1.0.alpha.14-1.x86_64.rpm
elif command -v apt-get >/dev/null 2>&1; then
  sudo apt-get update
  sudo apt install -y ./li9-s3-storage_1.1.0.alpha.14-1_amd64.deb
else
  echo "dnf or apt-get is required for this package install path." >&2
  exit 1
fi

ls -1 /opt/li9/s3-storage/bin
systemctl cat li9-s3-storage.target

3. Configure The Cluster

Generate one shared fencing token and one shared KMS master key once, store them in your secret manager, and reuse the same values on every node. The example below uses three storage nodes and starts the S3 gateway/control services on the first node.

export LI9_REGION=li9-mgm01
export LI9_ALLOWED_REGIONS=li9-mgm01,us-east-1
export LI9_SCHEME=http
export LI9_CONTROL_HOST=10.10.10.11
export LI9_GATEWAY_HOST=10.10.10.11
export LI9_STORAGE_NODE_URLS="${LI9_SCHEME}://10.10.10.11:9008,${LI9_SCHEME}://10.10.10.12:9008,${LI9_SCHEME}://10.10.10.13:9008"
export LI9_WRITE_QUORUM=2
export LI9_FENCING_TOKEN="$(openssl rand -base64 32 | tr -d '\n')"
export LI9_KMS_MASTER_KEY_B64="$(openssl rand -base64 32 | tr -d '\n')"

# Run on node 1:
sudo /opt/li9/s3-storage/bin/li9-s3-render-config \
  --node-host 127.0.0.1 \
  --control-host 127.0.0.1 \
  --gateway-host "${LI9_GATEWAY_HOST}" \
  --storage-node-urls "${LI9_SCHEME}://127.0.0.1:9008,${LI9_SCHEME}://10.10.10.12:9008,${LI9_SCHEME}://10.10.10.13:9008" \
  --write-quorum "${LI9_WRITE_QUORUM}" \
  --fencing-token "${LI9_FENCING_TOKEN}" \
  --kms-master-key-b64 "${LI9_KMS_MASTER_KEY_B64}" \
  --region "${LI9_REGION}" \
  --allowed-regions "${LI9_ALLOWED_REGIONS}"

# Run on node 2:
sudo /opt/li9/s3-storage/bin/li9-s3-render-config \
  --node-host 127.0.0.1 \
  --control-host "${LI9_CONTROL_HOST}" \
  --gateway-host "${LI9_GATEWAY_HOST}" \
  --storage-node-urls "${LI9_STORAGE_NODE_URLS}" \
  --write-quorum "${LI9_WRITE_QUORUM}" \
  --fencing-token "${LI9_FENCING_TOKEN}" \
  --kms-master-key-b64 "${LI9_KMS_MASTER_KEY_B64}" \
  --region "${LI9_REGION}" \
  --allowed-regions "${LI9_ALLOWED_REGIONS}"

# Run on node 3 with the same command as node 2.

Linux Configuration Properties

PropertyDescriptionAllowed Or Recommended Values
--node-hostAddress used by local services on this host.127.0.0.1 for local process wiring, or a routable host address when services are split.
--control-hostHost running control-api, identity, policy, bucket, metadata, KMS, and audit services.First node IP or DNS in the verified native layout.
--gateway-hostHost returned to S3 clients by the control API.Load balancer, DNS name, or first node IP for a lab install.
--storage-node-urlsComma-separated storage-node URLs used by object-service and healing-service.Example: three storage-node HTTP endpoints on port 9008.
--write-quorumNumber of storage-node writes required before an object write is committed.1 for single-node, 2 for three nodes.
--fencing-tokenShared token used to reject stale storage writes.Same 32-byte base64 value on all nodes.
--kms-master-key-b64Shared KMS/SSE master key.Same 32-byte base64 value on all nodes.
--regionDefault S3 region.Example: li9-mgm01; default is us-east-1.
--allowed-regionsComma-separated accepted S3 regions.Must include the default region.

4. Bootstrap, Join, And Start

# Run on every storage node:
sudo systemctl enable li9-s3-storage-node.target
sudo systemctl restart li9-s3@storage-node.service
sudo systemctl is-active --quiet li9-s3@storage-node.service

# Run on the control/gateway node:
sudo systemctl enable --now li9-s3-control.target li9-s3-gateway.target
curl -fsS "${LI9_SCHEME}://127.0.0.1:9001/v1/info"
curl -fsS "${LI9_SCHEME}://127.0.0.1:9000/v1/info"
curl -fsS "${LI9_SCHEME}://127.0.0.1:9008/v1/info"

5. Verify S3 And Node Failure

export S3_ENDPOINT="${LI9_SCHEME}://${LI9_GATEWAY_HOST}:9000"
export AWS_DEFAULT_REGION=li9-mgm01
PROVISION_JSON="$(curl -skS \
  -H 'Content-Type: application/json' \
  -d "{\"bucketName\":\"linux-install-check\",\"region\":\"${AWS_DEFAULT_REGION}\",\"issueCredential\":true}" \
  "${LI9_SCHEME}://${LI9_CONTROL_HOST}:9001/v1/s3-bucket-provisions")"
export AWS_ACCESS_KEY_ID="$(printf '%s' "${PROVISION_JSON}" | jq -r '.accessKeyId')"
export AWS_SECRET_ACCESS_KEY="$(printf '%s' "${PROVISION_JSON}" | jq -r '.secretKey')"
export AWS_SESSION_TOKEN="$(printf '%s' "${PROVISION_JSON}" | jq -r '.sessionToken // empty')"

aws --endpoint-url "${S3_ENDPOINT}" s3 mb s3://linux-install-check
dd if=/dev/urandom of=/tmp/linux-check.bin bs=1M count=64
aws --endpoint-url "${S3_ENDPOINT}" s3 cp /tmp/linux-check.bin s3://linux-install-check/linux-check.bin
aws --endpoint-url "${S3_ENDPOINT}" s3 cp s3://linux-install-check/linux-check.bin /tmp/linux-check-read.bin
sha256sum /tmp/linux-check.bin /tmp/linux-check-read.bin

# In another terminal, stop one node during client writes:
sudo systemctl stop li9-s3@storage-node.service
sudo systemctl start li9-s3@storage-node.service
curl -fsS "${LI9_SCHEME}://${LI9_CONTROL_HOST}:9009/v1/info" || sudo systemctl start li9-s3@healing-service.service
