Artifact Set

A disconnected installation is only repeatable when every artifact for the release is promoted together. Do not mirror only the operator image; the catalog, bundle, runtime images, documentation images, Helm chart, Linux packages, Windows packages, checksums, and SBOMs must come from the same release contract.

ArtifactDestinationValidation
Operator catalog and bundleInternal OCI registry used by OLM.skopeo inspect and CatalogSource pod readiness.
Runtime and documentation imagesInternal OCI registry pullable by cluster nodes.Digest match against release manifest.
Helm chartInternal Helm repository or OCI chart path.helm show chart works without internet.
RPM/DEB packagesInternal Linux package repository.Package checksums match and dependencies resolve offline.
MSI/ZIP packagesInternal Windows software share or deployment system.MSI and ZIP checksums match the release manifest.

Acceptance Criteria

Before starting installation, verify that a cluster node and an admin workstation can resolve the internal registry, trust its CA, and pull the selected image or package. For OpenShift, the CatalogSource must become Ready before creating Subscriptions.

export TARGET_IMAGE_REPO="registry.li9.internal/li9/li9-operators"
export LI9_RUNTIME_VERSION="1.1.0-alpha.15"

oc run li9-airgap-pullcheck \
  -n default \
  --rm -i --restart=Never \
  --image="${TARGET_IMAGE_REPO}:li9-s3-gateway-v${LI9_RUNTIME_VERSION}" \
  --command -- /bin/sh -c 'echo pull-ok'
oc get catalogsource -n openshift-marketplace li9-operators -o yaml
