Private Registry And Airgap

Private Registry And Airgap: Upgrade.

Upgrade safely with pre-checks, risk controls, rollout verification, and rollback awareness.

Upgrade

Airgap upgrades are two-step operations: promote artifacts first, then upgrade the selected installation type from the internal source.

RiskWhat Can HappenMitigation
Incomplete mirrorOperator or runtime pulls fail because a related image was missed.Mirror from the release manifest and verify every digest before approving rollout.
Mutable tagsTag references may point to different content after promotion.Prefer digest-pinned images and checksum-verified package artifacts.
Catalog mismatchCatalog points to bundle images that were not mirrored.Inspect bundle relatedImages before publishing the catalog.
Trust failureCluster nodes reject the internal registry CA or internal package repository trust chain.Distribute the registry CA and package repository keys before the upgrade window.
 
# Stage new artifacts
export TARGET_REGISTRY="registry.li9.internal/li9"
export TARGET_IMAGE_REPO="${TARGET_REGISTRY}/li9-operators"
export LI9_TARGET_OPERATOR_VERSION="1.1.0-alpha.40"
export LI9_TARGET_CHART_VERSION="1.1.0-alpha.15"
export LI9_TARGET_RUNTIME_PACKAGE_VERSION="1.1.0-alpha.14"
: "${INTERNAL_CHART_REPO:?set to your internal Helm chart repository URL}"

while read -r image; do
  tag="${image##*:}"
  skopeo copy --all "docker://${image}" "docker://${TARGET_IMAGE_REPO}:${tag}"
done < li9-s3-images-new-version.txt

# Update catalog source image
oc patch catalogsource -n openshift-marketplace li9-operators --type merge \
  -p "{\"spec\":{\"image\":\"${TARGET_IMAGE_REPO}:li9-s3-catalog-v${LI9_TARGET_OPERATOR_VERSION}\"}}"
oc wait pod -n openshift-marketplace -l olm.catalogSource=li9-operators --for=condition=Ready --timeout=5m

# Then upgrade through the selected installation page:
# Operator: approve/install new InstallPlan.
# Helm: helm upgrade from internal chart repo.
# Linux: dnf/apt upgrade from internal package repo.
# Windows: MSI/ZIP upgrade from internal package share.

Promotion Verification

Before upgrading a disconnected cluster or host fleet, verify the promoted release as an artifact set. The bundle, catalog, runtime images, Helm chart OCI artifact, RPM/DEB packages, MSI, ZIP, SBOMs, and checksums must all refer to the same release contract.

skopeo inspect "docker://${TARGET_IMAGE_REPO}:li9-s3-catalog-v${LI9_TARGET_OPERATOR_VERSION}"
helm repo add li9-s3-internal "${INTERNAL_CHART_REPO}"
helm repo update
helm pull li9-s3-internal/li9-s3-storage --version "${LI9_TARGET_CHART_VERSION}"
sha256sum -c checksums.txt --ignore-missing
export LI9_WINDOWS_MSI_PATH="./li9-s3-storage-windows-${LI9_TARGET_RUNTIME_PACKAGE_VERSION}.msi"
if [ -f "${LI9_WINDOWS_MSI_PATH}" ] && command -v pwsh >/dev/null 2>&1; then
  pwsh -NoProfile -Command 'Get-FileHash $env:LI9_WINDOWS_MSI_PATH -Algorithm SHA256'
fi

Rollback Guidance

Keep the previous internal catalog and package repositories until every target has passed post-upgrade validation. Rollback means switching the consumer back to the previous internal release source, not pulling from the public internet.
