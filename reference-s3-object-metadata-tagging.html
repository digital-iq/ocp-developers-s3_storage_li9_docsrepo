Reference / S3 API

Object metadata and tagging.

Metadata and tags attach searchable, policy-relevant, and lifecycle-relevant attributes to objects. Metadata is returned through object headers; tags are managed through the S3 tagging APIs and can participate in lifecycle and replication filters.

Common AWS CLI setup

Run this once for a bucket issued by the active platform control surface. On OpenShift, the values come from the generated S3Bucket or S3BucketAccess Secret. On Helm, Linux, or Windows, use the credentials issued by the runtime control API or administrator tooling.

export NS=li9-s3-runtime
export BUCKET_CR=app-data
export BUCKET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.bucketName}')"
export SECRET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.secretName}')"
export ENDPOINT="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ENDPOINT_URL}' | base64 -d)"
export AWS_ACCESS_KEY_ID="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ACCESS_KEY_ID}' | base64 -d)"
export AWS_SECRET_ACCESS_KEY="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_SECRET_ACCESS_KEY}' | base64 -d)"
export AWS_REGION="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_REGION}' 2>/dev/null | base64 -d || true)"
export AWS_REGION="${AWS_REGION:-us-east-1}"
export AWS_EC2_METADATA_DISABLED=true
li9s3() {
  aws --endpoint-url "$ENDPOINT" --region "$AWS_REGION" --no-verify-ssl s3api "$@"
}

For Helm, Linux, and Windows replace the exported values with the endpoint and credentials from that platform. The commands below intentionally use standard AWS CLI syntax after the setup.

Metadata Versus Tags

FeatureUse ForImportant Behavior
User metadataApplication annotations returned with object headers.Set at object write/copy time and stored with the object version.
Object tagsLifecycle, replication, classification, and authorization conditions.Can be updated independently and can be version-specific.
Bucket tagsAdministrative classification of buckets.Useful for reporting and ownership workflows.

Object Metadata And Tags

printf 'metadata
' > /tmp/metadata.txt
li9s3 put-object   --bucket "$BUCKET"   --key metadata/item.txt   --body /tmp/metadata.txt   --metadata owner=platform,environment=test   --tagging 'project=li9&class=demo'
li9s3 head-object --bucket "$BUCKET" --key metadata/item.txt --query Metadata
li9s3 get-object-tagging --bucket "$BUCKET" --key metadata/item.txt
li9s3 put-object-tagging --bucket "$BUCKET" --key metadata/item.txt --tagging 'TagSet=[{Key=project,Value=li9},{Key=class,Value=verified}]'
li9s3 get-object-tagging --bucket "$BUCKET" --key metadata/item.txt

Bucket Tags

li9s3 put-bucket-tagging --bucket "$BUCKET" --tagging 'TagSet=[{Key=owner,Value=platform},{Key=environment,Value=test}]'
li9s3 get-bucket-tagging --bucket "$BUCKET"

Validation Rules

  • Reserved metadata prefixes are controlled by the service and should not be used by applications.
  • Tag keys and values should follow AWS-compatible size and count limits.
  • Lifecycle and replication tag filters match exact tag key/value pairs.
