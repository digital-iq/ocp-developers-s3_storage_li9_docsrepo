OpenShift Operator

OpenShift Operator: Upgrade.

Upgrade safely with pre-checks, risk controls, rollout verification, and rollback awareness.

Upgrade

Operator upgrades can change the controller, CRD schema, runtime image defaults, and bundle metadata. Treat every production upgrade as a controlled rollout.

RiskWhat Can HappenMitigation
CRD schema changeExisting CRs may fail validation or require field changes.Run server-side dry-run against all CRs before approving the InstallPlan.
Runtime rolloutPods restart and may temporarily reduce write availability if quorum is already degraded.Confirm all conditions are healthy before upgrade and watch rollout one component at a time.
PVC expansionSome storage classes require pod restart before filesystem resize is visible.Let the operator perform ordered restarts and verify PVC capacity after each step.
Manual InstallPlanUpgrade waits until an admin approves it.Use Manual in production when change windows are required.
 
oc get s3storage -A
oc get installplan -n li9-operators
oc patch subscription -n li9-operators li9-s3-operator --type merge -p '{"spec":{"installPlanApproval":"Manual"}}'
oc get installplan -n li9-operators
# Review the pending InstallPlan name, then approve during the change window:
oc patch installplan -n li9-operators INSTALL_PLAN_NAME --type merge -p '{"spec":{"approved":true}}'
oc get csv -n li9-operators
oc rollout status deploy -n li9-operators -l app.kubernetes.io/name=li9-s3-operator --timeout=10m
oc get s3storage -A -o wide

Post-Upgrade Verification

Verify the operator, CRD schema, runtime rollout, and S3 client behavior before closing the change window. A successful upgrade has a healthy CSV, Ready S3Storage, no stuck InstallPlans, no degraded StatefulSets, and a passing AWS CLI smoke test from a workload namespace.

oc get csv,subscription,installplan -n li9-operators
oc get s3storage -A -o wide
oc get s3bucket,s3bucketaccess,s3bucketpolicy,s3bucketlifecycle,s3archivepolicy,s3replicationtarget,s3bucketreplication -A
oc get pods,pvc,svc,route -n li9-s3-runtime -l app.kubernetes.io/part-of=li9-s3
oc get events -A --field-selector type=Warning | grep -E 'li9-s3|storage.li9.com' || true

Rollback Guidance

Rollback is safe only when the previous operator version understands the current CRD schema and runtime data format. If the upgrade changed CRD fields, metadata format, encryption provider behavior, or archive placement, prefer forward-fix unless release notes explicitly mark the downgrade path as supported.
