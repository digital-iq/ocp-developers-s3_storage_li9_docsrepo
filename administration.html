Administration

Manage S3 tenants, buckets, policies, data lifecycle, and security controls.

Administration documents describe how platform teams expose Li9 S3 to application teams while keeping ownership boundaries clear. The operator owns Kubernetes resources; the runtime owns S3 protocol behavior and data protection.

Buckets and access

Use S3Bucket to declare a bucket and S3BucketAccess to issue scoped credentials. The access resource defines the exact actions and prefixes available to the application owner.

Recommended pattern

Create one bucket per workload boundary, then create separate access resources for writers, readers, and automation jobs.

Avoid

Do not reuse one broad credential across unrelated teams. That makes incident response and policy review harder.

Lifecycle policy and restore

Lifecycle policy belongs either on the bucket or in a reusable archive policy object when multiple buckets share the same retention and archive behavior.

FeatureAdministrative purposeOperator surface
ExpirationRemove old object versions or stale temporary data.S3BucketLifecycle or bucket lifecycle settings.
ArchiveMove colder data to an archive tier while keeping restore semantics visible.S3ArchivePolicy or bucket archive settings.
RestoreBring archived data back to the active tier for reads.Runtime API and status-backed background work.

Replication

Replication is a bucket-level feature for copying objects to another bucket or storage target. It is not the same as data-plane replication used for durability inside one Li9 S3 deployment.

Rule: use data protection settings for local durability; use bucket replication for copy workflows, migration, DR patterns, and cross-target synchronization.

Security and encryption

Security administration covers S3 access credentials, AWS-style bucket policy, TLS, KMS-style server-side encryption, object lock, and audit records.

ControlPurposeOperational signal
Bucket policyAllow or deny S3 actions by subject, resource, prefix, and conditions.Gateway authorization decision metrics.
TLSProtect client traffic and service-to-service traffic.Certificate expiry and handshake error metrics.
KMS-style SSEEncrypt data with platform-controlled key material.KMS operation errors and audit outbox lag.
AuditRecord S3 and KMS-relevant actions for investigation.Audit event rate and persist failures.
