Troubleshooting Guide

Start with CR status, managed resources, operator logs, and route/service health. Most failures are visible in status.conditions.

Fast Triage

SymptomFirst checkLikely owner
CR stays Progressingoc describe s3storage and managed workload readiness.Operator reconciliation or platform resources.

Operator Is Not Reconciling

oc get pods -n <operator-namespace> -l app.kubernetes.io/name=li9-s3-operator
oc logs -n <operator-namespace> deploy/li9-s3-controller-manager
oc get lease -n <operator-namespace> li9-s3-operator-leader -o yaml
  • Check that exactly one replica holds the leader lease.
  • Check RBAC for the affected namespace.
  • Check whether the CR is in a namespace watched by the operator.

S3Storage Is Not Ready

oc get s3storage -A
oc describe s3storage -n <namespace> <name>
oc get deploy,sts,pvc,svc,route -n <serviceNamespace>
  • Verify storage class names and PVC binding.
  • Verify node labels, tolerations, and failure-domain labels.
  • Check that the selected protection mode has enough nodes.

Documentation Site Is Not Available

oc get s3documentation -A
oc get deploy,svc,route -n <namespace> -l app.kubernetes.io/name=li9-s3-documentation
oc get s3documentation -n <namespace> <name> -o jsonpath='{.status.url}'
oc logs -n <namespace> deploy/<name>-docs -c runtime-docs
oc logs -n <namespace> deploy/<name>-docs -c operator-docs
  • If the Deployment is not ready, check image pull access for both docs images.
  • If the Service works but the Route does not, check route host conflicts and router admission.
  • If /runtime/ is unavailable but the root site works, check the runtime-docs container and the proxy environment in the operator-docs container.

Application Cannot Use Generated Credentials

oc get s3bucket,s3bucketaccess -n <namespace>
oc get secret -n <namespace> <secret-name> -o yaml
aws --endpoint-url <endpoint> s3api head-bucket --bucket <bucket>
  • Confirm endpoint, bucket, region, access key, secret key, and session token are mounted into the pod.
  • Check bucket policy and access actions.
  • Check gateway logs for SigV4, time-skew, or authorization errors.
