Upgrade

Upgrade operator, runtime services, charts, and native packages without losing data.

Upgrade documentation separates packaging from runtime behavior. Updating the operator changes reconciliation logic and desired images; updating runtime services changes the S3 data path. Both require compatibility checks.

Upgrade paths

PathUse whenRequired validation
Operator / OLMOpenShift operator-managed deployments.CSV succeeded, CRD conversion, status conditions, rollout health.
HelmChart-managed Kubernetes deployments.Values compatibility, rendered manifests, no operator conflict in namespace.
Linux packagesVM or bare-metal Linux clusters.Package upgrade, service restart order, disk path ownership, quorum.
Windows packagesWindows node deployments.MSI/ZIP install state, service restart order, firewall and TLS configuration.

Pre-upgrade checklist

  • Confirm all buckets and access resources are reconciled.
  • Confirm metadata quorum is healthy and healing backlog is empty or understood.
  • Export CRs, Helm values, or package configuration before changing versions.
  • Run AWS CLI smoke tests before and after the upgrade.
  • Monitor gateway errors, object commit failures, metadata latency, and healing work during rollout.
