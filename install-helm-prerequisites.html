Installation Mode Conflict Check

Operator, Helm, Linux package, and Windows package deployments are separate lifecycle models. Before installing Li9 S3 with Helm into a namespace, remove any operator-managed S3Storage runtime or host package deployment that owns the same endpoint, PVCs, Services, CRs, or storage paths.

Air-Gapped Prerequisites

Mirror both the chart and the container images before installing. Use a private Helm repository, a GitHub Pages equivalent inside the network, or an OCI chart in the internal registry.

export INTERNAL_REGISTRY="registry.li9.internal/li9/li9-operators"
: "${INTERNAL_CHART_REPO:?set to your internal Helm chart repository URL}"
export INTERNAL_REGISTRY_HOST="registry.li9.internal"
read -r -p "Registry user: " REGISTRY_USER
read -r -s -p "Registry password: " REGISTRY_PASSWORD
echo

helm registry login "${INTERNAL_REGISTRY_HOST}"
helm repo add li9-s3-internal "${INTERNAL_CHART_REPO}"
helm repo update
oc create secret docker-registry li9-s3-registry \
  --namespace li9-s3-helm \
  --docker-server="${INTERNAL_REGISTRY_HOST}" \
  --docker-username="${REGISTRY_USER}" \
  --docker-password="${REGISTRY_PASSWORD}"
oc secrets link default li9-s3-registry --for=pull -n li9-s3-helm
