Reference / S3 API

Checksums and payload integrity.

Li9 S3 validates client payload integrity, stores checksum metadata, and returns checksum information through AWS-compatible object attributes and read paths where supported.

Common AWS CLI setup

Run this once for a bucket issued by the active platform control surface. On OpenShift, the values come from the generated S3Bucket or S3BucketAccess Secret. On Helm, Linux, or Windows, use the credentials issued by the runtime control API or administrator tooling.

export NS=li9-s3-runtime
export BUCKET_CR=app-data
export BUCKET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.bucketName}')"
export SECRET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.secretName}')"
export ENDPOINT="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ENDPOINT_URL}' | base64 -d)"
export AWS_ACCESS_KEY_ID="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ACCESS_KEY_ID}' | base64 -d)"
export AWS_SECRET_ACCESS_KEY="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_SECRET_ACCESS_KEY}' | base64 -d)"
export AWS_REGION="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_REGION}' 2>/dev/null | base64 -d || true)"
export AWS_REGION="${AWS_REGION:-us-east-1}"
export AWS_EC2_METADATA_DISABLED=true
li9s3() {
  aws --endpoint-url "$ENDPOINT" --region "$AWS_REGION" --no-verify-ssl s3api "$@"
}

For Helm, Linux, and Windows replace the exported values with the endpoint and credentials from that platform. The commands below intentionally use standard AWS CLI syntax after the setup.

Integrity Controls

ControlPurposeWhere It Appears
Content-MD5Rejects corrupted payloads on write.Put object and multipart part uploads.
SigV4 payload hashProtects signed request bodies.AWS Signature Version 4 requests.
Flexible checksumsStores and returns CRC/SHA checksum metadata.put-object, head-object, object attributes.
Multipart aggregationPreserves integrity across multipart uploads.Complete multipart upload and object attributes.

SHA256 Example

printf 'checksum
' > /tmp/checksum.txt
li9s3 put-object --bucket "$BUCKET" --key checksums/sha256.txt --body /tmp/checksum.txt --checksum-algorithm SHA256
li9s3 head-object --bucket "$BUCKET" --key checksums/sha256.txt --checksum-mode ENABLED
li9s3 get-object-attributes   --bucket "$BUCKET"   --key checksums/sha256.txt   --object-attributes Checksum,ObjectSize,StorageClass

Negative Test

When a client supplies a checksum that does not match the payload, the service should reject the write rather than storing corrupted data.

printf 'bad-checksum
' > /tmp/bad-checksum.txt
aws --endpoint-url "$ENDPOINT" --region "$AWS_REGION" --no-verify-ssl s3api put-object   --bucket "$BUCKET"   --key checksums/bad.txt   --body /tmp/bad-checksum.txt   --content-md5 AAAAAAAAAAAAAAAAAAAAAA== || test "$?" -ne 0
