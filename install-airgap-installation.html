Installation

1. Create the release artifact list

Use digest-pinned images and checksum-verified package files from the published release.

2. Mirror container images

Copy runtime, operator, bundle, catalog, documentation, and client utility images into the internal registry.

3. Configure OpenShift image mirroring

Create ImageDigestMirrorSet or ImageTagMirrorSet CRs when the cluster should rewrite pulls automatically.

4. Create internal catalog source

Publish the mirrored Li9 catalog through the cluster-wide OLM catalog namespace.

5. Install from internal sources

Use the mirrored catalog for OLM, internal chart repository for Helm, and internal package repositories for Linux/Windows.

6. Verify internal-only operation

Inspect image references, pull events, and package sources.

1. Create Image And Artifact Lists

Use the public Quay and GitHub release locations as the connected upstream source, then mirror the same release set into the internal registry and package repositories used by the disconnected environment.

The connected Helm source is the Helm chart archive. Replace LI9_HELM_CHART_URL with an internal mirror URL after the chart is copied into your disconnected artifact store.

export LI9_OPERATOR_VERSION=1.1.0-alpha.40
export LI9_RUNTIME_VERSION=1.1.0-alpha.15
export LI9_CHART_VERSION=1.1.0-alpha.15
export LI9_IMAGE_REPO=quay.io/li9/li9-operators
: "${LI9_HELM_CHART_URL:?set to the Helm chart archive or internal mirror URL}"
export LI9_HELM_CHART_SHA256=28664fbc774e79a7c227de80acbcfba2b91c5fe457e664d92a74e4e0c78f6043

cat > li9-s3-images.txt <<EOF
${LI9_IMAGE_REPO}:li9-s3-catalog-v${LI9_OPERATOR_VERSION}
${LI9_IMAGE_REPO}:li9-s3-bundle-v${LI9_OPERATOR_VERSION}
${LI9_IMAGE_REPO}:li9-s3-manager-v${LI9_OPERATOR_VERSION}
${LI9_IMAGE_REPO}:li9-s3-operator-docs-v${LI9_OPERATOR_VERSION}
${LI9_IMAGE_REPO}:li9-s3-runtime-docs-v${LI9_RUNTIME_VERSION}

${LI9_IMAGE_REPO}:li9-s3-bucket-service-v${LI9_RUNTIME_VERSION}

printf '%s  %s\n' "${LI9_HELM_CHART_SHA256}" "li9-s3-storage-${LI9_CHART_VERSION}.tgz" | sha256sum -c -

2. Mirror Images

export TARGET_REGISTRY="registry.li9.internal/li9"
export TARGET_IMAGE_REPO="${TARGET_REGISTRY}/li9-operators"
export TARGET_AWS_CLI_REPO="${TARGET_REGISTRY}/aws-cli"

while read -r image; do
  tag="${image##*:}"
  if [ "${image}" = "public.ecr.aws/aws-cli/aws-cli:latest" ]; then
    target="${TARGET_AWS_CLI_REPO}:latest"
  else
    target="${TARGET_IMAGE_REPO}:${tag}"
  fi
  skopeo copy --all \
    "docker://${image}" \
    "docker://${target}"
done < li9-s3-images.txt

3. Create OpenShift Mirror CRs

cat <<EOF | envsubst | oc apply -f -
apiVersion: config.openshift.io/v1
kind: ImageDigestMirrorSet
metadata:
  name: li9-s3-digest-mirrors
spec:
  imageDigestMirrors:
    - source: quay.io/li9/li9-operators
      mirrors:
        - ${TARGET_IMAGE_REPO}
---
apiVersion: config.openshift.io/v1
kind: ImageTagMirrorSet
metadata:
  name: li9-s3-tag-mirrors
spec:
  imageTagMirrors:
    - source: quay.io/li9/li9-operators
      mirrors:
        - ${TARGET_IMAGE_REPO}
EOF
oc get imagedigestmirrorset,imagetagmirrorset | grep li9-s3

Mirror CR Properties

CRPropertyDescriptionAllowed Or Recommended Values

ImageDigestMirrorSetspec.imageDigestMirrors[].sourceOriginal source registry/repository for digest pulls.quay.io/li9/li9-operators.
ImageDigestMirrorSetspec.imageDigestMirrors[].mirrors[]Internal mirror locations.Internal registry repository reachable by cluster nodes.
ImageTagMirrorSetmetadata.nameCluster-scoped mirror rule name.Stable name such as li9-s3-tag-mirrors.
ImageTagMirrorSetspec.imageTagMirrors[].sourceOriginal source registry/repository for tag pulls.quay.io/li9/li9-operators. Prefer digest pulls for releases.
ImageTagMirrorSetspec.imageTagMirrors[].mirrors[]Internal mirror locations.Internal registry repository reachable by cluster nodes.

4. Create Internal CatalogSource

cat <<EOF | envsubst | oc apply -f -
apiVersion: operators.coreos.com/v1alpha1
kind: CatalogSource
metadata:
  name: li9-operators
  namespace: openshift-marketplace
spec:
  sourceType: grpc
  image: ${TARGET_IMAGE_REPO}:li9-s3-catalog-v${LI9_OPERATOR_VERSION}
  displayName: Li9 Operators
  publisher: Li9
EOF
oc wait pod -n openshift-marketplace -l olm.catalogSource=li9-operators --for=condition=Ready --timeout=5m

6. Verify Internal-Only Operation

oc get pods -A -l app.kubernetes.io/part-of=li9-s3 -o jsonpath='{range .items[*]}{.metadata.namespace}{" "}{.metadata.name}{" "}{range .spec.containers[*]}{.image}{" "}{end}{"\n"}{end}'
oc get events -A --field-selector type=Warning | grep -E 'li9-s3|ImagePull|ErrImagePull|BackOff' || true
oc get catalogsource -n openshift-marketplace li9-operators -o yaml | grep "${TARGET_IMAGE_REPO}"
helm repo list | grep li9-s3
dnf repolist 2>/dev/null | grep li9-s3 || true
apt-cache policy li9-s3-storage 2>/dev/null || true
