Custom Resource Examples

These examples show the operator-facing API shape for storage, bucket consumption, access control, data management, replication, and documentation exposure. Tune storage classes, node placement, image pull secrets, and route hosts for each cluster.

Apply examples in order: create S3Storage, wait for Ready, create S3Bucket, then attach access, policy, lifecycle, archive, or replication CRs as needed.

Full Distributed Storage With Erasure Coding

apiVersion: storage.li9.com/v1alpha1
kind: S3Storage
metadata:
  name: prod-s3
  namespace: li9-s3
spec:
  deletionPolicy: Retain
  region:
    default: li9-mgm01
    allowed:
      - li9-mgm01
      - us-east-1
  deployment:
    mode: FullDistributed
  exposure:
    route:
      enabled: false
  console:
    enabled: true
    rbacNamespace: app-team
    route:
      enabled: true
  storage:
    data:
      protection:
        type: ErasureCoding
        erasureCoding:
          dataShards: 2
          parityShards: 2
      placement:
        layout: PerNodePVC
        nodes: 4
        failureDomain:
          type: Node
      pvc:
        size: 100Gi
        storageClassName: ocs-storagecluster-ceph-rbd
    metadata:
      placement:
        type: ColocatedWithData
    archive:
      enabled: true
      pvc:
        size: 100Gi
        storageClassName: ocs-storagecluster-cephfs
  integrations:
    obc: true
    cosi: true

Single-Node Replicated Storage

apiVersion: storage.li9.com/v1alpha1
kind: S3Storage
metadata:
  name: single-node-s3
  namespace: li9-s3
spec:
  deployment:
    mode: FullDistributed
  region:
    default: li9-mgm01
    allowed:
      - li9-mgm01
  storage:
    data:
      protection:
        type: Replication
        replication:
          replicas: 1
      placement:
        layout: PerNodePVC
        nodes: 1
      pvc:
        size: 20Gi
        storageClassName: ocs-storagecluster-ceph-rbd
    metadata:
      placement:
        type: ColocatedWithData

Bucket And Access Secret

apiVersion: storage.li9.com/v1alpha1
kind: S3Bucket
metadata:
  name: app-data
  namespace: app-team
spec:
  bucketName: app-data
  region: li9-mgm01
  storageRef:
    name: prod-s3
    namespace: li9-s3
---
apiVersion: storage.li9.com/v1alpha1
kind: S3BucketAccess
metadata:
  name: app-data-writer
  namespace: app-team
spec:
  s3BucketRef: app-data
  secretName: app-data-writer
  access:
    effect: Allow
    actions:
      - s3:GetObject
      - s3:PutObject
      - s3:ListBucket
    resources:
      - bucket
      - objects
    prefixes:
      - uploads/

Bucket Policy

apiVersion: storage.li9.com/v1alpha1
kind: S3BucketPolicy
metadata:
  name: app-data-policy
  namespace: app-team
spec:
  s3BucketRef: app-data
  policy:
    Version: "2012-10-17"
    Statement:
      - Sid: AllowApplicationPrefix
        Effect: Allow
        Principal:
          AWS: "*"
        Action:
          - s3:GetObject
          - s3:PutObject
        Resource:
          - arn:aws:s3:::app-data/uploads/*
          - arn:aws:s3:::app-data

Lifecycle, Archive, And Replication

apiVersion: storage.li9.com/v1alpha1
kind: S3BucketLifecycle
metadata:
  name: app-data-lifecycle
  namespace: app-team
spec:
  s3BucketRef: app-data
  document: |
    <LifecycleConfiguration>
      <Rule>
        <ID>expire-temp-uploads</ID>
        <Status>Enabled</Status>
        <Filter>
          <Prefix>tmp/</Prefix>
        </Filter>
        <Expiration>
          <Days>7</Days>
        </Expiration>
      </Rule>
    </LifecycleConfiguration>
---
apiVersion: storage.li9.com/v1alpha1
kind: S3ArchivePolicy
metadata:
  name: app-data-archive
  namespace: app-team
spec:
  s3BucketRef: app-data
  storageClass: DEEP_ARCHIVE
  prefixes:
    - reports/
  transitionAfterDays: 30
  expireAfterDays: 365
---
apiVersion: storage.li9.com/v1alpha1
kind: S3ReplicationTarget
metadata:
  name: remote-dr
  namespace: app-team
spec:
  endpoint: REMOTE_S3_ENDPOINT
  region: li9-dr-1
  addressingStyle: PathStyle
  credentialsSecretRef:
    name: remote-dr-s3-credentials
---
apiVersion: storage.li9.com/v1alpha1
kind: S3BucketReplication
metadata:
  name: app-data-replication
  namespace: app-team
spec:
  s3BucketRef: app-data
  rules:
    - id: replicate-critical-prefix
      prefix: critical/
      destination:
        type: Local
        bucketName: app-data-copy
    - id: replicate-audit-to-dr
      prefix: audit/
      destination:
        type: Global
        bucketName: app-data-dr
        globalTargetRef:
          name: remote-dr

Use product terms such as archive and deep archive in documentation and UI. Values such as DEEP_ARCHIVE are S3 API storage-class values kept for AWS CLI and SDK compatibility.
