Runtime service catalog

Service responsibilities, boundaries, and operating signals.

API and Provisioning

s3-gateway

S3 entry point

Business value
Gives teams one S3-compatible endpoint for AWS CLI, SDKs, backup tools, logging stacks, and application pods.
Use when
Workloads need bucket/object APIs, multipart upload, object tags, metadata, object lock, checksums, and signed requests.
Do not use when
The requirement is an AWS-managed integration rather than an S3 protocol capability.
Architecture
Validates SigV4, delegates authorization, reads bucket metadata, calls object APIs, and emits durable audit/access-log events.
request latency4xx/5xx rateSigV4 failuresmultipart errors
control-api

Internal provisioning API

Business value
Lets operator, OBC/COSI, Helm automation, and platform tooling create buckets and credentials consistently.
Use when
Kubernetes CRs or platform workflows need a stable API that is not exposed to normal application traffic.
Do not use when
Application code only needs normal S3 object operations.
Architecture
Fronts bucket, identity, endpoint, and access provisioning while keeping controller workflows out of the public S3 edge.
provision latencyreconcile errorssecret writes

Security and Governance

identity-service

Credential and subject registry

Business value
Separates S3 credentials from web/operator authentication and supports bucket-scoped access.
Use when
Buckets need generated access keys, secret keys, session tokens, TTLs, or subject cleanup.
Do not use when
Credentials are expected to be fully owned by an external identity provider with no Li9 mapping.
Architecture
Persists identity state, supports generated Secrets, and handles expired credential cleanup.
credential countexpired cleanupauth lookup latency
policy-service

AWS-style authorization policy

Business value
Enforces bucket access with familiar S3 policy semantics and explicit allow/deny decisions.
Use when
Teams need action scoping, resource scoping, prefixes, conditions, and predictable denied-request behavior.
Do not use when
The requirement is AWS account IAM federation rather than local S3 policy evaluation.
Architecture
Evaluates request context from the gateway and returns a decision without owning object data.
allow/deny ratepolicy parse errorsdecision latency
kms-service

Local KMS-style encryption API

Business value
Supports server-side encryption without requiring AWS KMS.
Use when
Data must be encrypted with generated or existing key material controlled by the platform.
Do not use when
Policy requires an external HSM or enterprise KMS that has not been integrated and tested.
Architecture
Manages local master key material, data-key operations, and durable encryption audit delivery.
key opsencrypt/decrypt errorsaudit outbox lag
audit-service

Security and access audit sink

Business value
Gives security and operations teams a durable record of S3 and KMS-relevant activity.
Use when
Bucket access, credential use, encryption events, and gateway operations must be investigated later.
Do not use when
Audit data is intentionally sent only to an external SIEM and local persistence is disabled.
Architecture
Persists events to PVC-backed state and receives durable outbox deliveries from gateway and KMS services.
event rateoutbox retriespersist errors

Bucket, Metadata, and Data

bucket-service

Bucket catalog and configuration

Business value
Centralizes bucket lifecycle, versioning, tags, replication, policy, archive, website, and object-lock settings.
Use when
Bucket configuration must be consistent across S3 APIs, operator CRs, and background jobs.
Do not use when
Only raw filesystem object placement is needed with no S3 bucket semantics.
Architecture
Delegates durable persistence to the metadata plane and exposes bucket-specific control APIs.
bucket countconfig errorscontrol latency
metadata-service

Object and bucket metadata

Business value
Makes bucket state and object manifests durable independently from gateway replicas.
Use when
Listings, versions, restore status, lifecycle state, object lock, and shard manifests must survive restarts.
Do not use when
The deployment is intentionally stateless and external metadata is authoritative.
Architecture
Uses PVC-backed Raft/KV/WAL metadata, either dedicated or colocated with data in per-node deployments.
raft healthWAL errorsmetadata latency
object-service

Object placement and S3 semantics

Business value
Owns the durability behavior that turns S3 requests into protected object storage.
Use when
Objects need erasure coding, replication, archive placement, restore behavior, checksums, or multipart assembly.
Do not use when
Object bytes should bypass product metadata, authorization, and durability rules.
Architecture
Writes shards or replicas, commits manifests, manages archive tiering, and exposes internal APIs used by gateway and healing.
write latencyread latencycommit failuresmultipart state
storage-node

Shard persistence service

Business value
Isolates PVC or native path access so only one service owns shard files and disk-level durability probes.
Use when
Li9 owns the storage plane through per-node PVCs, shared PVCs, or VM/native disk paths.
Do not use when
The data plane is external and object-service is configured for an external backend.
Architecture
Validates fencing headers, writes shard files durably, exposes shard read/write/delete APIs and health information.
disk capacityfencing rejectsshard write errors

Background Automation

healing-service

Repair and policy execution

Business value
Keeps stored data healthy and applies lifecycle, replication, archive, restore, and cleanup policies.
Use when
The platform needs automatic recovery after node return or scheduled object management.
Do not use when
All background object management is delegated to an external service.
Architecture
Scans metadata, compares shard state, retries durable outboxes, and performs idempotent repair work.
repair queuerepaired objectspolicy lagrestore errors
