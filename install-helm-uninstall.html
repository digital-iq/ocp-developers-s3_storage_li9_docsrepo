Helm Chart

Helm Chart: Uninstall.

Remove the installation in dependency order and verify that managed resources are gone.

Uninstall Checklist

Uninstall the Helm release first, then decide whether retained PVCs and generated Secrets should stay available for forensic recovery, backup export, or reinstallation. Helm does not own business data-retention policy, so destructive cleanup must be a separate approved step.

Deleting PVCs is destructive. Do it only when the data owner has approved storage removal.
StepPurposeExpected Result
Record current valuesKeep the exact installed configuration for reinstall or audit.A local final-values.yaml file exists.
Uninstall releaseRemove Kubernetes workloads and chart-owned resources.No Li9 S3 pods or services remain.
Review retained dataSeparate safe application cleanup from destructive PVC cleanup.PVC and Secret decisions are documented.
Remove namespaceOnly for disposable environments with no retained data.The namespace disappears without finalizer loops.
 
export LI9_NS=li9-s3-helm
helm status li9-s3 -n "${LI9_NS}"
helm get values li9-s3 -n "${LI9_NS}" --all > final-values.yaml
helm uninstall li9-s3 -n "${LI9_NS}"
oc get all,route,servicemonitor,prometheusrule -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3

# Destructive cleanup only:
# oc delete pvc -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3
# oc delete secret,configmap -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3

oc get pods -n "${LI9_NS}" | grep li9-s3 || true

Verification

The release is fully removed when Helm no longer lists it, no Li9 S3 workloads remain, and any remaining PVCs are intentionally retained. If the uninstall hangs, inspect finalizers and events before deleting PVCs manually.

helm list -n "${LI9_NS}" | grep li9-s3 || true
oc get pods,svc,route -n "${LI9_NS}" | grep li9-s3 || true
oc get pvc -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3
oc get events -n "${LI9_NS}" --field-selector type=Warning | grep li9-s3 || true
