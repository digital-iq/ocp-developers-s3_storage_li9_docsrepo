Helm Chart

Helm Chart: Installation.

Run the production installation procedure with copy-paste commands and field references.

Installation

1. Add the chart repository

Use public chart source for online installs or internal chart source for air-gapped installs.

2. Create values file

Declare image source, gateway exposure, storage topology, metadata placement, and observability.

3. Render and install

Review manifests, run server-side validation, then install the release with helm upgrade --install --wait.

4. Verify runtime and S3

Check pods, PVCs, services, route or ingress, metrics objects, and AWS CLI behavior from outside the runtime pods.

1. Add Chart Repository

Use the public Helm repository for connected installs, or replace LI9_HELM_REPO with your internal chart repository URL.

: "${LI9_HELM_REPO:?set to the public Helm repository or internal chart repository URL}"
export LI9_CHART_VERSION=1.1.0-alpha.15
export LI9_CHART_URL="${LI9_HELM_REPO}/li9-s3-storage-${LI9_CHART_VERSION}.tgz"
export LI9_CHART_SHA256=28664fbc774e79a7c227de80acbcfba2b91c5fe457e664d92a74e4e0c78f6043

curl -fL "${LI9_CHART_URL}" -o "/tmp/li9-s3-storage-${LI9_CHART_VERSION}.tgz"
printf '%s  %s\n' "${LI9_CHART_SHA256}" "/tmp/li9-s3-storage-${LI9_CHART_VERSION}.tgz" | sha256sum -c -

helm repo add li9-s3 "${LI9_HELM_REPO}"
helm repo update
helm show chart li9-s3/li9-s3-storage --version "${LI9_CHART_VERSION}"

# Air-gapped
helm repo add li9-s3 "${INTERNAL_CHART_REPO}"
helm repo update
helm show chart li9-s3/li9-s3-storage --version "${LI9_CHART_VERSION}"

2. Create Values File

cat > values-li9-s3.yaml <<'EOF'
images:
  repository: quay.io/li9/li9-operators
  tag: ""
imagePullSecrets: []
openshift:
  enabled: true

region:
  default: li9-mgm01
  allowed:
    - li9-mgm01
    - us-east-1

gateway:
  routeEnabled: true

runtime:
  replicas: 2
  storageNodeReplicas: 3
  writeQuorum: 2
storage:
  topology:
    mode: PerNodePVC
  objectBackend:
    type: StorageNodeErasure
    volumeSize: 20Gi
    storageClassName: ocs-storagecluster-ceph-rbd
  metadata:
    mode: DedicatedPVC
    volumeSize: 5Gi
    storageClassName: ocs-storagecluster-ceph-rbd
EOF

Helm Values Reference

ValueDescriptionAllowed Or Recommended Values
LI9_CHART_URLDirect chart artifact used by this install guide.Helm chart archive.
images.repositoryRepository containing runtime images.quay.io/li9/li9-operators online; internal registry path for air-gapped installs.
images.tagRuntime image version suffix. Empty uses chart appVersion.Empty resolves this chart to runtime image tags such as quay.io/li9/li9-operators:li9-s3-gateway-v1.1.0-alpha.15.
imagePullSecretsPull secrets attached to runtime pods.Empty list for public images; list of Secret names for private registries.
openshift.enabledRenders OpenShift-specific resources such as Route, ServiceMonitor, PrometheusRule, and NetworkPolicy.true for OpenShift installs.
region.defaultDefault S3 region for buckets created without a location constraint.Example: li9-mgm01. If omitted, the chart uses us-east-1.
region.allowedExplicit region allow-list served by this Helm release.Must include region.default when set. Empty means only the default region is allowed.
gateway.routeEnabledCreates OpenShift Route.false by default; true when route exposure is desired.
runtime.replicasReplica count for replicated control/gateway services.2 or more for production.
runtime.storageNodeReplicasNumber of storage-node pods.Must satisfy the chosen write quorum and data-protection target.
runtime.writeQuorumMinimum storage-node writes needed to commit object payloads.For three storage nodes, 2 tolerates one node outage.
storage.topology.modeData PVC topology.PerNodePVC or SharedPVC.
storage.objectBackend.typeObject payload backend.StorageNodeErasure for Li9-owned storage nodes; Filesystem for S3-over-PVC.
storage.objectBackend.volumeSizeObject data PVC size.Kubernetes quantity such as 20Gi or 1Ti.
storage.objectBackend.storageClassNameObject data PVC StorageClass.Any class compatible with the selected topology.
storage.metadata.modeMetadata placement.DedicatedPVC or SharedDataPVC.
storage.metadata.volumeSizeMetadata PVC size.Kubernetes quantity such as 5Gi.
storage.metadata.storageClassNameMetadata PVC StorageClass.Low-latency class preferred.

3. Render And Install

export LI9_NS=li9-s3-helm
helm template li9-s3 li9-s3/li9-s3-storage \
  --namespace "${LI9_NS}" \
  --version "${LI9_CHART_VERSION}" \
  --values values-li9-s3.yaml > rendered-li9-s3.yaml

kubectl apply --dry-run=server -f rendered-li9-s3.yaml

helm upgrade --install li9-s3 li9-s3/li9-s3-storage \
  --namespace "${LI9_NS}" \
  --create-namespace \
  --version "${LI9_CHART_VERSION}" \
  --values values-li9-s3.yaml \
  --wait \
  --timeout 20m

4. Verify Runtime And S3

helm status li9-s3 -n "${LI9_NS}"
oc get pods,pvc,svc,route,servicemonitor,prometheusrule -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3
oc wait pod -n "${LI9_NS}" -l app.kubernetes.io/part-of=li9-s3 --for=condition=Ready --timeout=10m

export S3_ENDPOINT="$(printf '%s://%s' https "$(oc get route -n "${LI9_NS}" li9-s3-gateway -o jsonpath='{.spec.host}' 2>/dev/null || true)")"
export AWS_DEFAULT_REGION=li9-mgm01
export LI9_CONTROL_SCHEME=https
mkdir -p /tmp/li9-s3-helm-certs
oc -n "${LI9_NS}" extract secret/li9-s3-internal-tls --keys=tls.crt,tls.key --to=/tmp/li9-s3-helm-certs --confirm
oc -n "${LI9_NS}" port-forward svc/control-api 18443:8080 &
PF_PID=$!
trap 'kill "${PF_PID}" 2>/dev/null || true' EXIT
until curl -skS --cert /tmp/li9-s3-helm-certs/tls.crt --key /tmp/li9-s3-helm-certs/tls.key "${LI9_CONTROL_SCHEME}://127.0.0.1:18443/v1/info" >/dev/null; do sleep 1; done

PROVISION_JSON="$(curl -skS --cert /tmp/li9-s3-helm-certs/tls.crt --key /tmp/li9-s3-helm-certs/tls.key \
  -H 'Content-Type: application/json' \
  -d "{\"bucketName\":\"helm-install-check\",\"region\":\"${AWS_DEFAULT_REGION}\",\"issueCredential\":true}" \
  "${LI9_CONTROL_SCHEME}://127.0.0.1:18443/v1/s3-bucket-provisions")"
export AWS_ACCESS_KEY_ID="$(printf '%s' "${PROVISION_JSON}" | jq -r '.accessKeyId')"
export AWS_SECRET_ACCESS_KEY="$(printf '%s' "${PROVISION_JSON}" | jq -r '.secretKey')"
export AWS_SESSION_TOKEN="$(printf '%s' "${PROVISION_JSON}" | jq -r '.sessionToken // empty')"

aws --endpoint-url "${S3_ENDPOINT}" --region "${AWS_DEFAULT_REGION}" s3api head-bucket --bucket helm-install-check
aws --endpoint-url "${S3_ENDPOINT}" --region "${AWS_DEFAULT_REGION}" s3 cp values-li9-s3.yaml s3://helm-install-check/values.yaml
aws --endpoint-url "${S3_ENDPOINT}" --region "${AWS_DEFAULT_REGION}" s3 cp s3://helm-install-check/values.yaml -
