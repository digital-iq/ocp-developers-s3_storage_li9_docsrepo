Linux Packages

Linux Packages: Uninstall.

Remove the installation in dependency order and verify that managed resources are gone.

Uninstall Checklist

Stop services on one host at a time, verify the cluster can still serve traffic, then remove the package. Preserve data, metadata, WAL, certificate, and audit directories unless the data owner explicitly approves destructive cleanup.

Removing packages does not always remove data. Deleting /var/lib/li9/s3-storage is destructive and cannot be undone without backup.
StepPurposeExpected Result
Health checkConfirm the remaining nodes can maintain quorum and data protection.Gateway, control API, and storage-node /v1/info endpoints are healthy before removal.
Stop servicePrevent writes during package removal on the target host.No Li9 S3 process listens on the host ports.
Remove packageRemove binaries and systemd units.The package manager no longer lists li9-s3-storage.
Retain or wipe dataPreserve local state for recovery unless removal is approved.Data directories are either retained and labeled or wiped intentionally.
 
sudo systemctl stop li9-s3-storage.target li9-s3-control.target li9-s3-gateway.target li9-s3-storage-node.target || true
sudo systemctl disable li9-s3-storage.target li9-s3-control.target li9-s3-gateway.target li9-s3-storage-node.target || true

# RPM:
sudo dnf remove -y li9-s3-storage

# DEB:
sudo apt-get remove -y li9-s3-storage

systemctl status 'li9-s3@*.service' --no-pager || true
sudo ss -lntp | grep -E '900[0-9]|9010' || true
rpm -q li9-s3-storage || dpkg -l li9-s3-storage || true

# Destructive cleanup only:
# sudo rm -rf /var/lib/li9/s3-storage /etc/li9/s3-storage /var/log/li9/s3-storage

Cluster Verification

After each host is removed, run the same S3 client smoke test from an external workstation, a test pod, or another remaining Li9 S3 node. A production uninstall should not be treated as complete until clients either fail over to the remaining cluster or the service is intentionally shut down everywhere.

aws --endpoint-url "${S3_ENDPOINT}" s3api list-buckets || true
export LI9_SCHEME="${LI9_SCHEME:-http}"
export LI9_CONTROL_HOST="${LI9_CONTROL_HOST:-10.10.10.11}"
export LI9_GATEWAY_HOST="${LI9_GATEWAY_HOST:-10.10.10.11}"
curl -fsS "${LI9_SCHEME}://${LI9_GATEWAY_HOST}:9000/v1/info" || true
curl -fsS "${LI9_SCHEME}://${LI9_CONTROL_HOST}:9001/v1/info" || true
curl -fsS "${LI9_SCHEME}://${LI9_CONTROL_HOST}:9008/v1/info" || true
