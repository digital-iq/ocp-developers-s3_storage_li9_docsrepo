Private Registry And Airgap

Private Registry And Airgap: Uninstall.

Remove the installation in dependency order and verify that managed resources are gone.

Uninstall Checklist

Remove Li9 S3 from each target installation first. Remove mirror rules, catalog sources, internal registry artifacts, package repositories, and trust bundles only after no remaining cluster or host depends on that release.

Do not delete internal mirrored images or package repositories until all clusters and hosts using that release have been upgraded or removed.
StepPurposeExpected Result
Inventory consumersFind clusters, Helm installs, Linux hosts, and Windows hosts still pulling from the internal mirror.No active consumer points at the release being removed.
Uninstall productUse the installation-specific uninstall page first.No Li9 S3 runtime workload depends on mirrored images.
Remove cluster mirror configDelete OpenShift mirror and catalog objects when they are no longer shared.Catalog and IDMS/ITMS objects are gone or intentionally retained for other Li9 operators.
Clean internal artifactsRemove image, chart, RPM/DEB, and MSI/ZIP artifacts only from retired release channels.Internal registry and package repository state matches the approved retention policy.
 
# Remove Li9 S3 install through its install-specific page first.
# Then remove airgap-only OpenShift mirror configuration if no remaining Li9 operator uses it:
oc delete catalogsource -n openshift-marketplace li9-operators
oc delete imagedigestmirrorset li9-s3-digest-mirrors
oc delete imagetagmirrorset li9-s3-tag-mirrors

# Verify no Li9 S3 pods still depend on internal images:
oc get pods -A -o jsonpath='{range .items[*]}{.metadata.namespace}{" "}{.metadata.name}{" "}{range .spec.containers[*]}{.image}{" "}{end}{"\n"}{end}' | grep li9-s3 || true
oc get events -A --field-selector type=Warning | grep -E 'li9-s3|ImagePull|ErrImagePull|BackOff' || true

Verification

A clean airgap uninstall has no Li9 S3 consumers and no broken catalog or image-pull events. Keep shared li9-operators catalog infrastructure when it also serves other Li9 products.
