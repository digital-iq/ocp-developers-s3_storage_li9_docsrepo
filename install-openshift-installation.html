OpenShift Operator

OpenShift Operator: Installation.

Run the production installation procedure with copy-paste commands and field references.

Installation

1. Create the operator namespace

This namespace hosts OLM resources and the controller deployment. Runtime resources can be created in tenant namespaces.

2. Register the catalog

Create a CatalogSource from the online or mirrored catalog image.

3. Subscribe to the operator

Install Li9 S3 Storage from OpenShift Software Catalog or create the equivalent OperatorGroup and Subscription.

4. Wait for operator readiness

Verify the package manifest, CSV, CRDs, and controller deployment before creating runtime CRs.

5. Create S3Storage

Choose data protection, metadata placement, PVC sizes, gateway exposure, and observability.

6. Create bucket, access, and optional docs

Create S3Bucket and S3BucketAccess resources for applications, and create S3Documentation when operators need in-cluster documentation.

7. Verify with AWS CLI

Validate from a separate client pod so the test matches a real application flow.

1. Create Namespace

oc new-project li9-operators || oc project li9-operators
oc label namespace li9-operators app.kubernetes.io/part-of=li9-operators --overwrite

2. Create CatalogSource

Use the public release catalog image for OLM discovery. The bundle image is listed and checked explicitly because the catalog resolves this bundle when the Subscription installs the operator. For the public online release, do not set spec.secrets on the CatalogSource and do not create a Li9 registry pull secret in the customer cluster.

export LI9_OPERATOR_VERSION=1.1.0-alpha.40
export LI9_IMAGE_REPO=quay.io/li9/li9-operators
export LI9_BUNDLE_IMAGE="${LI9_IMAGE_REPO}:li9-s3-bundle-v${LI9_OPERATOR_VERSION}"
export LI9_CATALOG_IMAGE="${LI9_IMAGE_REPO}:li9-s3-catalog-v${LI9_OPERATOR_VERSION}"

oc image info "${LI9_BUNDLE_IMAGE}" --filter-by-os=linux/amd64
oc image info "${LI9_CATALOG_IMAGE}" --filter-by-os=linux/amd64

cat <<'EOF' | envsubst | oc apply -f -
apiVersion: operators.coreos.com/v1alpha1
kind: CatalogSource
metadata:
  name: li9-operators
  namespace: openshift-marketplace
spec:
  sourceType: grpc
  image: ${LI9_CATALOG_IMAGE}
  displayName: Li9 Operators
  publisher: Li9
  updateStrategy:
    registryPoll:
      interval: 30m
EOF
oc wait pod -n openshift-marketplace -l olm.catalogSource=li9-operators --for=condition=Ready --timeout=5m

CatalogSource Properties

PropertyDescriptionAllowed Or Recommended Values
metadata.nameCatalogSource name referenced by Li9 operator Subscriptions.li9-operators. Keep this S3-neutral because the same catalog will publish multiple Li9 operators.
metadata.namespaceNamespace where OLM reads catalog sources.openshift-marketplace for cluster-visible catalogs.
spec.sourceTypeCatalog protocol.grpc.
spec.imageIndex image containing Li9 operator packages, including Li9 S3 Storage.quay.io/li9/li9-operators:li9-s3-catalog-v1.1.0-alpha.40 for the current public release, or a mirrored internal registry image for disconnected clusters.
spec.secretsOptional pull secret list for private catalog images.Omit for the public Li9 Quay catalog. Use only when installing from a private internal registry that requires authentication.
LI9_BUNDLE_IMAGEBundle image resolved by the catalog for this release.quay.io/li9/li9-operators:li9-s3-bundle-v1.1.0-alpha.40. Verify pull access before creating the Subscription.
spec.displayNameName displayed in OperatorHub.Li9 Operators.
spec.publisherVendor displayed in OperatorHub.Li9.
spec.updateStrategy.registryPoll.intervalHow often OLM checks for catalog updates.Example: 30m. Increase in tightly controlled production clusters.

3. Subscribe To The Operator

Use the OpenShift Console path when an administrator wants to install the operator interactively from the catalog. Use the CLI path when the installation is driven by GitOps, automation, or a reviewed change record. Both paths create the same OLM Subscription and result in the same installed CSV.

Console Installation

  1. Open the OpenShift Console in the Administrator perspective.
  2. Go to Operators, then open OperatorHub or Software Catalog, depending on the console version.
  3. Search for Li9 S3 Storage Operator. If it is not listed, verify the li9-operators CatalogSource from step 2 is Ready in openshift-marketplace.
  4. Open the operator tile and click Install.
  5. Select the alpha update channel for the current published release.
  6. Select li9-operators as the installed namespace. If the namespace is not offered, create it first with step 1.
  7. Select the approval strategy. Use Automatic for hands-off upgrades, or Manual when production changes require InstallPlan review.
  8. Click Install and wait until the operator appears under Installed Operators with status Succeeded.
Console FieldValue To SelectWhy It Matters
CatalogLi9 Operators from openshift-marketplace.This is the catalog registered in step 2 and it contains the Li9 S3 package.
OperatorLi9 S3 Storage Operator.This installs the controller that reconciles S3Storage, bucket, access, lifecycle, replication, policy, and documentation CRs.
Update channelalpha for the current published release line.The channel controls which CSV version OLM installs and upgrades to.
Installed namespaceli9-operators.The operator deployment and Subscription live in the shared Li9 operator namespace.
Installation modeCluster-scoped installation.The operator must reconcile Li9 S3 CRs across namespaces.
Approval strategyAutomatic for hands-off rollout or Manual for reviewed production changes.Manual approval lets administrators inspect InstallPlans before OLM applies upgrades.
VersionSelect the newest CSV shown for the chosen channel unless a change record pins another version.The installed CSV version should match the release approved for the environment.

After clicking Install, keep the Console open until the status changes from installing to Succeeded. Then open Installed Operators, select Li9 S3 Storage Operator, and confirm that the provided APIs include the Li9 S3 CRDs before creating runtime resources.

CLI Installation

cat <<'EOF' | oc apply -f -
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: li9-operators
  namespace: li9-operators
spec: {}
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: li9-s3-operator
  namespace: li9-operators
spec:
  channel: alpha
  installPlanApproval: Automatic
  name: li9-s3-operator
  source: li9-operators
  sourceNamespace: openshift-marketplace
EOF
oc get subscription -n li9-operators li9-s3-operator
oc get installplan -n li9-operators

5. Create S3Storage

oc new-project li9-s3-runtime || oc project li9-s3-runtime
cat <<'EOF' | oc apply -f -
apiVersion: storage.li9.com/v1alpha1
kind: S3Storage
metadata:
  name: li9-s3
  namespace: li9-s3-runtime
spec:
  deletionPolicy: Retain
  region:
    default: li9-mgm01
    allowed:
      - li9-mgm01
      - us-east-1
  exposure:
    route:
      enabled: false
  console:
    enabled: true
    rbacNamespace: li9-s3-runtime
    route:
      enabled: true
  storage:
    data:
      protection:
        type: Replication
        replication:
          replicas: 3
      placement:
        layout: PerNodePVC
        nodes: 3
        failureDomain:
          type: Node
      pvc:
        size: 20Gi
        storageClassName: ocs-storagecluster-ceph-rbd
    metadata:
      placement:
        type: ColocatedWithData
      pvc:
        size: 5Gi
        storageClassName: ocs-storagecluster-ceph-rbd
  observability:
    metrics:
      serviceMonitor:
        enabled: true
      prometheusRule:
        enabled: true
EOF
oc wait s3storage -n li9-s3-runtime li9-s3 --for=condition=Ready --timeout=20m
oc get pods,pvc,svc -n li9-s3-runtime -l app.kubernetes.io/part-of=li9-s3

S3Storage Properties

PropertyDescriptionAllowed Or Recommended Values
spec.deletionPolicyControls data handling when the CR is removed.Retain keeps PVC data; Delete performs destructive cleanup and removes runtime data.
spec.region.defaultDefault S3 region for buckets that do not set spec.region.Use a valid SigV4 token such as li9-mgm01. If omitted, the operator uses us-east-1 for AWS CLI parity.
spec.region.allowedExplicit region allow-list for this storage instance.Must include spec.region.default when set. If omitted or empty, only the default region is allowed.
spec.exposure.route.enabledCreates an OpenShift Route for the S3 endpoint.false by default; true only when external client access through Route is approved.
spec.exposure.route.hostOptional fixed external S3 DNS name.Set when DNS is pre-reserved. If omitted and Route is enabled, the operator uses the OpenShift-admitted Route host once available.
status.endpointEffective endpoint written to bucket/access Secrets.External Route endpoint when Route host is explicit or admitted; otherwise the internal s3-gateway Service endpoint in the service namespace.
spec.console.enabledDeploys the OpenShift OAuth-protected bucket browser.false by default. Set true when human users should browse buckets through OpenShift identity.
spec.console.route.enabledCreates an OpenShift Route for the bucket browser.Usually true with console enabled. The browser route is separate from the S3 gateway route.
spec.console.rbacNamespaceNamespace where the browser checks s3buckets/objects RBAC.Defaults to the runtime namespace. Use a tenant namespace when bucket-object permissions are managed there.
spec.storage.data.protection.typeData protection engine.Replication for one or more replicas; ErasureCoding for shard-based durability; External for an externally durable backend.
spec.storage.data.protection.replication.replicasReplica count when protection type is replication.1 for explicit single-node or edge installs, 3 or more for production.
spec.storage.data.protection.erasureCoding.dataShardsNumber of required data shards in erasure coding.Integer greater than zero. Example: 2.
spec.storage.data.protection.erasureCoding.parityShardsNumber of parity shards that can be lost without data loss.Integer greater than zero. Example: 2.
spec.storage.data.placement.layoutPVC topology for data.PerNodePVC for RWO node-local PVCs; SharedPVC for RWX shared PVCs; TieredPVC for separated active/archive placement.
spec.storage.data.placement.nodesNumber of storage nodes to schedule.Must match durability design. Example: 1, 3, 4, or more.
spec.storage.data.placement.failureDomain.typeFailure-domain strategy.Node, TopologyLabel, or Storage. External backends default to Storage.
spec.storage.data.pvc.sizeRequested PVC size for data.Kubernetes quantity such as 20Gi, 1Ti.
spec.storage.data.pvc.storageClassNameStorage class for data PVCs.Any cluster StorageClass compatible with the chosen layout.
spec.storage.metadata.placement.typeMetadata placement.ColocatedWithData places metadata with data; DedicatedPVC uses separate metadata PVCs.
spec.storage.metadata.pvc.sizeMetadata PVC size when metadata has a PVC.Kubernetes quantity. Size for KV/WAL metadata growth and compaction.
spec.storage.metadata.pvc.storageClassNameStorage class for metadata PVCs.Low-latency class preferred. Can match data class if co-located.
spec.observability.metrics.serviceMonitor.enabledCreates ServiceMonitor resources.true when user workload Prometheus is enabled; otherwise false.
spec.observability.metrics.prometheusRule.enabledCreates PrometheusRule resources.true for production alerting; false for minimal installs.

6. Create Bucket, Access, And Optional Docs

cat <<'EOF' | oc apply -f -
apiVersion: storage.li9.com/v1alpha1
kind: S3Bucket
metadata:
  name: app-logs
  namespace: li9-s3-runtime
spec:
  storageRef:
    name: li9-s3
---
apiVersion: storage.li9.com/v1alpha1
kind: S3BucketAccess
metadata:
  name: app-logs-writer
  namespace: li9-s3-runtime
spec:
  s3BucketRef: app-logs
  access:
    effect: Allow
    actions:
      - s3:ListBucket
      - s3:GetObject
      - s3:PutObject
      - s3:DeleteObject
    prefixes:
      - ""
---
apiVersion: storage.li9.com/v1alpha1
kind: S3Documentation
metadata:
  name: li9-s3-docs
  namespace: li9-s3-runtime
spec:
  exposure:
    route:
      enabled: true
      tls: true
EOF
oc get s3bucket,s3bucketaccess,s3documentation -n li9-s3-runtime

OpenShift Console And Bucket Browser UI

When spec.console.enabled is true, the operator creates an OAuth-protected bucket browser. Grant users access with OpenShift RBAC on the s3buckets/objects subresource before asking them to open the Route.

cat <<'EOF' | oc -n li9-s3-runtime apply -f -
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: app-logs-browser-reader
rules:
  - apiGroups: ["storage.li9.com"]
    resources: ["s3buckets/objects"]
    resourceNames: ["app-logs"]
    verbs: ["list", "get"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: app-logs-browser-reader
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: Role
  name: app-logs-browser-reader
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: alice@example.com
EOF
oc -n li9-s3-runtime get route s3-console
oc -n li9-s3-runtime auth can-i list s3buckets.storage.li9.com/app-logs --subresource=objects --as=alice@example.com

S3Bucket Properties

PropertyDescriptionAllowed Or Recommended Values
metadata.nameKubernetes object name and default bucket name.DNS-compatible bucket name such as app-logs.
metadata.namespaceTenant namespace that owns the bucket request.Any namespace watched by the cluster-scoped operator.
spec.storageRef.nameTarget S3Storage instance.Name of an existing S3Storage resource.
spec.storageRef.namespaceOptional storage namespace when bucket lives elsewhere.Omit when bucket and storage are in the same namespace; set for cross-namespace consumption when allowed by policy.

S3BucketAccess Properties

PropertyDescriptionAllowed Or Recommended Values
metadata.nameAccess request name and generated Secret name.Stable application-oriented name, for example app-logs-writer.
spec.s3BucketRefSame-namespace S3Bucket resource to issue access for.Name of an existing S3Bucket. Use spec.bucketName with spec.storageRef only for direct bucket-name access.
spec.secretNameOptional generated Secret name.Omit to use the access CR name.
spec.access.effectManaged policy statement effect for this credential.Allow or Deny. Default is Allow.
spec.access.actionsAllowed S3 actions for generated credentials.Common values: s3:ListBucket, s3:GetObject, s3:PutObject, s3:DeleteObject. Use least privilege.
spec.access.resourcesOptional explicit AWS-style S3 ARN resources.Omit when resources can be derived from bucket and prefixes.
spec.access.prefixesObject key prefixes used to derive object ARNs.Use "" for the whole bucket or a prefix such as logs/.

PropertyDescriptionAllowed Or Recommended Values

spec.exposure.route.enabledCreates an OpenShift Route for the docs site.true to expose through Route; false for in-cluster Service only.
spec.exposure.route.tlsEnables Route TLS behavior.true for HTTPS routes; false only when TLS is terminated by an approved upstream component.

7. Verify With AWS CLI

export LI9_NS=li9-s3-runtime
export AWS_ACCESS_KEY_ID="$(oc get secret -n "${LI9_NS}" app-logs-writer -o jsonpath='{.data.AWS_ACCESS_KEY_ID}' | base64 -d)"
export AWS_SECRET_ACCESS_KEY="$(oc get secret -n "${LI9_NS}" app-logs-writer -o jsonpath='{.data.AWS_SECRET_ACCESS_KEY}' | base64 -d)"
export AWS_SESSION_TOKEN="$(oc get secret -n "${LI9_NS}" app-logs-writer -o jsonpath='{.data.AWS_SESSION_TOKEN}' 2>/dev/null | base64 -d || true)"
export AWS_DEFAULT_REGION="$(oc get secret -n "${LI9_NS}" app-logs-writer -o jsonpath='{.data.AWS_DEFAULT_REGION}' | base64 -d)"
export S3_ENDPOINT="$(oc get secret -n "${LI9_NS}" app-logs-writer -o jsonpath='{.data.AWS_ENDPOINT_URL}' | base64 -d)"
oc run -n "${LI9_NS}" aws-cli --rm -it --restart=Never --image=public.ecr.aws/aws-cli/aws-cli:latest --env AWS_ACCESS_KEY_ID --env AWS_SECRET_ACCESS_KEY --env AWS_SESSION_TOKEN --env AWS_DEFAULT_REGION --env S3_ENDPOINT --command -- sh -c '
aws --endpoint-url "${S3_ENDPOINT}" s3 ls
printf "operator install check\n" > /tmp/check.txt
aws --endpoint-url "${S3_ENDPOINT}" s3 cp /tmp/check.txt s3://app-logs/install-check.txt
aws --endpoint-url "${S3_ENDPOINT}" s3 cp s3://app-logs/install-check.txt -
'
