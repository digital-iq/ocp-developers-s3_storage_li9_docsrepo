Windows Packages

Windows Packages: Upgrade.

Upgrade safely with pre-checks, risk controls, rollout verification, and rollback awareness.

Upgrade

Windows upgrades replace binaries and restart local Li9 S3 processes. Upgrade one node at a time and keep the cluster observable during the rollout.

RiskWhat Can HappenMitigation
Process restartMSI or ZIP upgrade stops and restarts local Li9 S3 processes.Upgrade one node at a time and verify S3/control endpoints between nodes.
Configuration mismatchNew binary may reject old env values.Re-render service env files with Render-Li9S3Config.ps1 before restart.
Metadata migrationMetadata changes may not be reversible.Back up metadata and WAL paths before upgrade.
Firewall policy driftSecurity policy may remove required ports.Re-check firewall rules after upgrade.
 
$Li9TargetVersion = "1.1.0-alpha.14"
$Li9Scheme = "http"
$Li9ControlHost = "10.20.10.11"
$Li9GatewayHost = "10.20.10.11"
$Li9StorageNodeUrls = "${Li9Scheme}://10.20.10.11:9008,${Li9Scheme}://10.20.10.12:9008,${Li9Scheme}://10.20.10.13:9008"

$InstallDir = "C:\Program Files\Li9\S3Storage"
if (-not (Test-Path $InstallDir)) { $InstallDir = "C:\Li9\S3Storage" }
Invoke-WebRequest "${Li9Scheme}://${Li9GatewayHost}:9000/v1/info" -UseBasicParsing
Invoke-WebRequest "${Li9Scheme}://${Li9ControlHost}:9001/v1/info" -UseBasicParsing
& "$InstallDir\scripts\Stop-Li9S3Services.ps1"

# MSI upgrade
msiexec.exe /i "C:\Temp\li9-s3-storage-windows-$Li9TargetVersion.msi" /qn /norestart
$InstallDir = "C:\Program Files\Li9\S3Storage"

# ZIP upgrade
if (Test-Path "C:\Li9\S3Storage") {
  Rename-Item C:\Li9\S3Storage "C:\Li9\S3Storage.previous.$(Get-Date -Format yyyyMMddHHmmss)"
  Expand-Archive "C:\Temp\li9-s3-storage-windows-$Li9TargetVersion.zip" C:\Li9\S3Storage -Force
  $InstallDir = "C:\Li9\S3Storage"
}

& "$InstallDir\scripts\Render-Li9S3Config.ps1" `
  -NodeHost "127.0.0.1" `
  -ControlHost $Li9ControlHost `
  -GatewayHost $Li9GatewayHost `
  -StorageNodeUrls $Li9StorageNodeUrls `
  -WriteQuorum 2 `
  -Region li9-mgm01 `
  -AllowedRegions "li9-mgm01,us-east-1"
& "$InstallDir\scripts\Start-Li9S3Services.ps1" -InstallDir $InstallDir -Services @("storage-node")
& "$InstallDir\scripts\Start-Li9S3Services.ps1" -InstallDir $InstallDir -Services @("control-api","identity-service","policy-service","bucket-service","metadata-service","kms-service","audit-service","object-service","s3-gateway")
Invoke-WebRequest "${Li9Scheme}://${Li9GatewayHost}:9000/v1/info" -UseBasicParsing
Invoke-WebRequest "${Li9Scheme}://${Li9ControlHost}:9001/v1/info" -UseBasicParsing

Post-Upgrade Verification

After each Windows node upgrade, confirm service status, event logs, firewall state, cluster membership, and an external S3 client smoke test. Continue only when the upgraded node has rejoined cleanly and no metadata quorum warning remains.

Get-Process | Where-Object { $_.Path -like "*Li9*S3Storage*" }
Get-EventLog -LogName Application -Newest 50
Get-NetFirewallRule | Where-Object { $_.DisplayName -like "Li9 S3*" }
Invoke-WebRequest "${Li9Scheme}://${Li9GatewayHost}:9000/v1/info" -UseBasicParsing
Invoke-WebRequest "${Li9Scheme}://${Li9ControlHost}:9001/v1/info" -UseBasicParsing
aws --endpoint-url $env:S3_ENDPOINT s3api list-buckets

Rollback Guidance

MSI rollback restores binaries, not data format. Use rollback only when the previous runtime version is explicitly compatible with the upgraded metadata and object layout. For ZIP installs, keep the previous directory until the upgraded node passes health checks and the external S3 smoke test.
