Reference / S3 API

Range and conditional reads.

Range and conditional requests let clients resume downloads, validate caches, read multipart sections, and avoid transferring unchanged data. Li9 S3 should preserve AWS-compatible status behavior for these read paths.

Common AWS CLI setup

Run this once for a bucket issued by the active platform control surface. On OpenShift, the values come from the generated S3Bucket or S3BucketAccess Secret. On Helm, Linux, or Windows, use the credentials issued by the runtime control API or administrator tooling.

export NS=li9-s3-runtime
export BUCKET_CR=app-data
export BUCKET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.bucketName}')"
export SECRET="$(oc -n "$NS" get s3bucket "$BUCKET_CR" -o jsonpath='{.status.secretName}')"
export ENDPOINT="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ENDPOINT_URL}' | base64 -d)"
export AWS_ACCESS_KEY_ID="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_ACCESS_KEY_ID}' | base64 -d)"
export AWS_SECRET_ACCESS_KEY="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_SECRET_ACCESS_KEY}' | base64 -d)"
export AWS_REGION="$(oc -n "$NS" get secret "$SECRET" -o jsonpath='{.data.AWS_REGION}' 2>/dev/null | base64 -d || true)"
export AWS_REGION="${AWS_REGION:-us-east-1}"
export AWS_EC2_METADATA_DISABLED=true
li9s3() {
  aws --endpoint-url "$ENDPOINT" --region "$AWS_REGION" --no-verify-ssl s3api "$@"
}

For Helm, Linux, and Windows replace the exported values with the endpoint and credentials from that platform. The commands below intentionally use standard AWS CLI syntax after the setup.

ControlUse ForExpected Result
RangePartial object download.Returns only requested bytes with partial-content semantics.
If-MatchRead only when the ETag matches.Returns object or precondition failure.
If-None-MatchCache validation.Returns not-modified/precondition behavior when unchanged.
partNumberRead a specific multipart object part.Returns the selected part where multipart metadata is available.
Response header overridesSigned downloads with specific content headers.Returns overridden response headers without changing stored metadata.

Range And ETag Example

printf '0123456789abcdef
' > /tmp/range.txt
li9s3 put-object --bucket "$BUCKET" --key range/item.txt --body /tmp/range.txt
ETAG="$(li9s3 head-object --bucket "$BUCKET" --key range/item.txt --query ETag --output text)"
li9s3 get-object --bucket "$BUCKET" --key range/item.txt --range bytes=0-3 /tmp/range-part.txt
cat /tmp/range-part.txt
li9s3 get-object --bucket "$BUCKET" --key range/item.txt --if-match "$ETAG" /tmp/range-if-match.txt
li9s3 head-object --bucket "$BUCKET" --key range/item.txt --if-none-match "$ETAG" || test "$?" -ne 0

Verification

  • Compare the returned range with the expected byte slice.
  • Confirm matching ETag requests succeed and non-matching preconditions fail.
  • Confirm missing objects return AWS-style missing-object responses and do not mask backend outages as missing data.
